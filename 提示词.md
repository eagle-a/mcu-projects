## 历史问题与修复

### 1. 显示内容镜像（已修复）
**原因**：`epd_driver.c` 的 `epd_init_display_sequence` 混用了参考代码两种初始化模式：
- 用了 `EPD_HW_Init_180`（180° 旋转）的 `Data Entry Mode = 0x02`
- 却用了 `EPD_HW_Init_GUI`（镜像）的 RAM 地址范围（未反转）

**修复**（`@/home/zm/daily-dev/eink_clock/main/epd_driver.c`）：
- `0x11` Data Entry Mode：`0x02` → `0x01`（与 GUI/镜像模式一致）
- `epd_hw_init_fast`：补全缺失的 Data Entry + RAM 地址/计数器设置
- `0x22` Display Update Control：`0xCF`（4灰度值）→ `0xF7`（正常全刷）

### 2. 大/中字体显示异常（已修复）
**原因**：`font.c` 中 `font_large_digits` 和 `font_medium_digits` 的 bitmap 数据损坏（数字 2、3 等不是正常字形，出现奇怪的沙漏形状）。同时 `font_draw_large_digit` 的索引 bug（`byte_idx = row * 4` 应为 `row * 2`）。small 字体数据正确，所以底部时间日期显示正常。

**修复**（`@/home/zm/daily-dev/eink_clock/main/font.c:234-268`）：
- `font_draw_large_digit`：弃用损坏的 bitmap，改为 small 字体 2× 放大（每像素 → 2×2 块）
- `font_draw_medium_digit`：改为 small 字体 1.5× 最近邻缩放

## 参考要点（SSD1683 / GDEY042T81）

**Data Entry Mode (0x11)**：
- bit 2 (AM)：0=X方向，1=Y方向
- bit 1 (ID1)：Y 方向 0=dec，1=inc
- bit 0 (ID0)：X 方向 0=dec，1=inc

参考代码三种初始化：
| 模式 | Driver Output | Data Entry | RAM X | RAM Y |
|---|---|---|---|---|
| `EPD_HW_Init`（正常） | `0x00` | — | `0→49` | `299→0` |
| `EPD_HW_Init_GUI`（镜像） | `0x01` | `0x01` | `0→49` | `299→0` |
| `EPD_HW_Init_180`（180°） | — | `0x02` | `49→0`（反转） | `0→299`（反转） |
